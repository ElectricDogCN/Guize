# Never Rules — 归泽・Guize

## 1. 使用原则

Never Rules 记录已经识别出的高风险错误模式。任何 Agent、开发人员和自动化流程都不得违反。发现新的重复性错误后，必须先修复问题，再更新本文件和演进日志。

## 2. 数据与资产

1. **永不**将源路径当作逻辑资产唯一身份。
2. **永不**因单个来源删除而直接删除逻辑资产。
3. **永不**在未验证完整性的情况下把副本标记为 `VERIFIED`。
4. **永不**把 ATS 命中当作完整文件已缓存或已备份。
5. **永不**把普通完整缓存当作正式保留副本。
6. **永不**因内容哈希相同而覆盖不同来源的权限、标签、路径或保留策略。
7. **永不**在未备份和未审批时执行破坏性数据库迁移。
8. **永不**只依赖 OpenSearch 或 Milvus 保存权威业务状态。
9. **永不**将关键关系仅存入 JSONB。
10. **永不**跳过内容版本和来源版本的区分。

## 3. 权限与安全

11. **永不**在返回搜索标题、摘要或缩略图后才检查权限。
12. **永不**让衍生内容绕过原资产权限。
13. **永不**在 Git、日志、任务参数或错误响应中存储 Secrets 明文。
14. **永不**允许插件直连核心业务数据库。
15. **永不**允许匿名用户触发高成本转码、冷恢复或商业 AI 调用。
16. **永不**使用长期有效且不可撤销的公开媒体 URL。
17. **永不**把文件扩展名当作真实 MIME 类型。
18. **永不**自动解压未知压缩包而不做路径穿越、递归和体积限制。
19. **永不**让公网管理入口绕过限流、审计和异常登录检测。
20. **永不**使用灾难恢复密钥进行日常登录。

## 4. API、事件与任务

21. **永不**让长任务同步阻塞 HTTP 请求直至完成。
22. **永不**实现没有幂等性的重试任务。
23. **永不**假设事件只投递一次。
24. **永不**修改已发布 API 的语义而不升级版本或提供兼容期。
25. **永不**把本地化文本作为稳定错误判断依据；使用错误码。
26. **永不**在事件消费者中先产生不可逆副作用再记录处理状态。
27. **永不**让 Temporal Workflow 依赖非确定性运行时代码而不封装为 Activity。
28. **永不**在 LiteFlow Node 中直接运行长时间 FFmpeg、ASR 或备份任务。
29. **永不**在未记录任务输入版本的情况下覆盖已有 AI 或媒体结果。
30. **永不**将“请求已接收”当作“任务成功”。

## 5. 媒体、缓存与存储

31. **永不**在磁盘低于安全水位时继续非必要下载、转码或预热。
32. **永不**无配额地创建临时媒体文件。
33. **永不**使用未受限的 FFmpeg 协议和路径输入。
34. **永不**在未保存编码参数、工具版本和哈希时发布正式 Rendition。
35. **永不**为同一资产重复执行可复用的昂贵转码或 AI 处理。
36. **永不**删除唯一可恢复副本。
37. **永不**将同一聚合入口下的多个路径误认为独立故障域。
38. **永不**在回源带宽逼近预算时继续普通离线任务。
39. **永不**让 ATS 与完整缓存使用同一套状态和淘汰语义。
40. **永不**让缓存清理任务拥有删除正式副本的权限。

## 6. AI

41. **永不**只以 HTTP 200 判断 AI 能力质量合格。
42. **永不**在模型、Prompt 或分块策略变化后跳过回归测试。
43. **永不**让 AI 自动扩大公开范围、修改权限或删除正式数据。
44. **永不**将生成式缩略图伪装成真实关键帧。
45. **永不**把低置信度识别结果写成已确认事实。
46. **永不**在未做数据策略检查时向商业 API 发送敏感媒体。
47. **永不**允许 AI 自行突破流量、Token 或费用硬预算。
48. **永不**覆盖人工修订而不保留版本与审计记录。
49. **永不**使用未经许可证审查的模型权重进入生产。
50. **永不**让不同 Embedding 模型的向量混入同一版本空间而不记录模型版本。

## 7. 工程与发布

51. **永不**直接在主分支修改。
52. **永不**在需求、设计或契约未更新时实施行为变化。
53. **永不**通过删除测试、跳过测试或放宽断言修复门禁失败。
54. **永不**使用可变 Tag 作为生产镜像唯一定位。
55. **永不**部署未验证 Digest 或签名的镜像。
56. **永不**在数据库迁移前跳过备份。
57. **永不**重新构建已经通过测试的发布候选后再部署生产；必须晋升同一制品。
58. **永不**在没有回滚方法时发布高风险变更。
59. **永不**把 POC 结论直接当作生产结论。
60. **永不**将 Agent 的口头说明当作测试证据。

## 8. 文档

61. **永不**让代码、配置、契约和文档长期不一致。
62. **永不**在文档中声称尚未实测的性能数据。
63. **永不**将“推荐组件”写成不可替换的永久依赖而不记录 ADR。
64. **永不**为机器契约创建中英文两套 URL、字段或枚举标识。
65. **永不**删除历史决策记录来掩盖方案变化。
