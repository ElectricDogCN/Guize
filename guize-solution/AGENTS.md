# AGENTS.md — 归泽・Guize Agent 工程规则

## 1. 适用范围

本文件适用于在归泽项目中执行需求分析、设计、编码、测试、配置、部署、审查、修复和文档维护的所有 AI Agent，包括但不限于 Codex、Trae Solo、Trae Work 及其子 Agent。

本文件是强制规则，不是建议。任何任务与本文件冲突时，必须停止修改并报告冲突。

## 2. 总体目标

Agent 的目标不是“尽快产出代码”，而是：

1. 在冻结的产品和架构边界内完成任务；
2. 保证每个修改可验证、可追溯、可回滚；
3. 不破坏资产、权限、缓存、备份和任务一致性；
4. 不将未经验证的假设伪装为事实；
5. 让单人审查者能依据契约、测试和证据作出发布决定。

## 3. 权威信息优先级

发生冲突时，按以下顺序处理：

1. 已批准的需求规格；
2. 已批准的 API、事件、数据 Schema 契约；
3. 已批准的 ADR；
4. 系统和模块设计文档；
5. `AGENTS.md`；
6. `rules/never-rules.md`；
7. 当前任务说明；
8. 代码现状；
9. Agent 自行推断。

如果代码与已批准契约冲突，不得默认以代码为准。先记录冲突，再按任务权限修复或提交决策。

## 4. 开始任务前的强制步骤

每个任务必须先建立任务上下文：

```text
任务 ID
目标
业务背景
输入
输出
修改范围
禁止修改范围
关联需求
关联设计
关联契约
验收标准
验证命令
风险
回滚方式
```

开始编码前必须：

- 阅读相关需求、设计、ADR、契约和 Never Rules；
- 搜索现有实现、测试和配置；
- 确认模块所有权和数据库表所有权；
- 确认是否涉及破坏性 API、数据迁移、权限、Secrets、删除或公开；
- 为不确定项建立显式假设，不得隐式猜测；
- 先更新规格或契约，再进行实现。

## 5. 分支与提交

### 5.1 分支

每个任务必须使用独立分支：

```text
feat/<issue>-<short-name>
fix/<issue>-<short-name>
docs/<issue>-<short-name>
refactor/<issue>-<short-name>
chore/<issue>-<short-name>
```

禁止直接在主分支修改。

### 5.2 提交

提交必须：

- 单一目的；
- 可构建或明确说明中间状态；
- 引用 Issue；
- 描述行为变化；
- 不包含 Secrets、临时文件、模型权重和大型测试产物；
- 相关文档、测试和迁移与代码同提交或按可审查顺序提交。

推荐格式：

```text
feat(asset): add source-object relocation inference

Issue: GUIZE-123
Evidence: evidence/GUIZE-123/
```

## 6. 代码边界

### 6.1 Java 模块化单体

- 模块禁止直接访问其他模块 Repository；
- 模块禁止直接修改其他模块拥有的表；
- 模块通信使用公开接口、领域事件、Outbox 或只读查询契约；
- 禁止创建全局“大而全”Service；
- 核心业务不得依赖 Controller、ORM 实体或第三方 SDK 类型；
- 核心领域对象与 API DTO、持久化实体必须分离。

### 6.2 Python 能力服务

- Python 服务不得直连平台核心 PostgreSQL；
- 任务输入和输出必须符合版本化契约；
- 大文件通过受限对象引用、临时签名地址或受控挂载传递；
- 模型、Prompt、参数、运行环境和结果必须可追溯；
- 长任务必须幂等、可重试、支持取消或安全中断；
- 不得在日志中输出媒体敏感内容、Token 或凭据。

### 6.3 插件

- 插件不得直连核心数据库；
- 必须具有 Manifest、版本、能力和权限声明；
- 重型、不可信或第三方插件必须独立容器；
- 插件失败不能拖垮控制面；
- 插件 API 必须有超时、重试、熔断和健康检查；
- 插件必须通过契约测试。

## 7. 数据库和迁移

- PostgreSQL 是权威数据库；
- 核心关系使用规范化表，不得仅保存在 JSONB；
- Flyway 迁移必须只前进、可审查、可在备份后执行；
- 破坏性迁移必须提供 expand/migrate/contract 方案；
- 禁止未经审批直接删除列、表、索引或历史数据；
- 每次迁移必须有集成测试和回滚/恢复说明；
- 任务、权限、资产、版本、副本、审计不可通过无事务的多步更新维持一致性；
- Outbox 事件必须与业务事务同事务写入。

## 8. API 和事件

- URL、字段、枚举和错误码使用唯一英文机器标识；
- OpenAPI 和开发者文档提供中英文说明；
- 外部和内部 API 均版本化；
- 破坏性变更必须增加主版本或提供兼容层；
- 长任务立即返回任务 ID；
- 支持幂等键、状态查询、取消、暂停、恢复和错误详情；
- 每个响应包含 `traceId`；
- 事件使用统一 Envelope；
- 事件消费者必须幂等；
- 禁止依赖消息“只会投递一次”。

## 9. 权限和安全

涉及以下内容时必须增加专项测试：

- 用户、角色和 ACL；
- 匿名访问；
- 资产发布；
- 衍生数据访问；
- 搜索召回过滤；
- Secrets；
- 文件下载和签名地址；
- 管理员登录；
- 高风险操作；
- 数据导出；
- 备份和恢复。

安全规则：

- 权限过滤必须在召回或对象读取前执行；
- 衍生内容继承原资产权限；
- 管理员密码登录不得降低高风险操作的二次确认；
- 任何凭据只保存 Secret 引用；
- 公开文件必须符合 MIME、内容探测和安全策略；
- 禁止路径穿越、任意文件读取和开放代理；
- 所有外部输入必须验证；
- 默认拒绝未知能力和未知权限。

## 10. 文件、媒体和缓存

- ATS 缓存不等于完整文件副本；
- 普通完整缓存不等于正式保留；
- 缓存删除不得影响资产元数据、正式副本和保留状态；
- 删除源对象不得自动删除逻辑资产；
- 只有验证完成的副本才能标记 `VERIFIED`；
- 媒体任务必须记录输入哈希、输出哈希、编码参数和工具版本；
- FFmpeg 命令必须限制输入、协议、路径和资源；
- 临时目录必须有配额、生命周期和清理机制；
- 所有大文件任务必须支持断点、恢复或安全重跑。

## 11. AI 能力

- 不得仅凭模型返回成功判定质量合格；
- 必须保存模型、版本、Prompt、参数和输入版本；
- 模型或 Prompt 变更必须运行固定样本回归；
- AI 结果不得直接修改权威元数据，必须经过校验或发布流程；
- 低置信度结果必须标记；
- 商业 API 调用必须受预算、权限和数据策略控制；
- 不得自动公开、删除或降低安全策略；
- 生成式缩略图必须与真实媒体缩略图区分。

## 12. 测试门禁

任务完成至少需要：

- 单元测试；
- 相关集成测试；
- API/事件契约测试；
- 权限和越权测试；
- 数据迁移测试（如适用）；
- 状态机和失败路径测试；
- 静态分析和格式检查；
- Secrets 扫描；
- 文档链接和示例验证；
- 任务专项验收脚本。

发布候选还需要：

- 端到端测试；
- 性能和容量测试；
- 故障注入；
- 恢复测试；
- 安全扫描；
- 镜像签名和 Digest 验证；
- SBOM 与许可证检查。

## 13. 证据

每个任务必须在 `evidence/<task-id>/` 保存或引用：

```text
summary.md
commands.txt
test-results/
screenshots/
api-samples/
migration-report/
performance/
security/
rollback-verification/
```

证据必须能够由审查者复现。不得只写“测试通过”。

## 14. Agent 可执行与不可执行操作

### 可自动执行

- 读取代码与文档；
- 建立任务计划；
- 修改独立分支；
- 运行本地测试、静态分析和模拟；
- 生成配置草案；
- 生成数据库迁移草案；
- 生成部署 Bundle；
- 生成诊断和修复建议。

### 必须人工批准

- 合并主分支；
- 生产发布；
- 数据库正式迁移；
- Secrets 修改；
- 权限策略发布；
- 删除或覆盖正式数据；
- 调低安全水位；
- 开放公网管理能力；
- 高风险插件升级；
- 灾难恢复；
- 备份密钥操作。

## 15. Ralph/Wiggum 式可重复验证循环

长任务必须按循环推进：

```text
读取规格
→ 观察现状
→ 提出最小变更
→ 实施
→ 运行验证
→ 分析失败
→ 修正
→ 重复验证
→ 记录证据
→ 人工审查
```

禁止在验证失败后通过删除测试、放宽断言、屏蔽异常或跳过门禁宣称完成。

## 16. 结束任务的标准

Agent 只有在以下条件全部满足时才能标记完成：

- 需求和契约一致；
- 修改范围未越界；
- 测试门禁通过；
- 失败路径验证；
- 文档同步；
- 证据完整；
- 回滚方法可执行；
- 未解决项明确；
- 无新增 Never Rule 违规；
- 人工审查所需信息完整。

否则状态必须是 `PARTIAL`、`BLOCKED` 或 `NEEDS_REVIEW`。
