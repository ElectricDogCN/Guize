# Never Rules 演进日志

## 2026-07-21 — V1 基线建立

### 新增

- 建立资产、来源、版本、Rendition 和 Replica 的身份边界规则。
- 建立 ATS 缓存、完整缓存和正式副本的隔离规则。
- 建立权限前置过滤、衍生内容继承和 Secrets 规则。
- 建立 Temporal、LiteFlow、事件幂等和长任务规则。
- 建立媒体、FFmpeg、存储水位和流量预算规则。
- 建立 AI 质量、许可证、模型版本和人工修订规则。
- 建立 Git、迁移、镜像签名、测试证据和文档同步规则。

### 触发背景

归泽 V1 由 Agent 主导开发、单人审查。若仅依赖人工逐行审查，无法覆盖数据一致性、安全、媒体任务、插件和供应链风险，因此将已经确认的禁止性约束固化为自动检查和审查基线。

### 后续维护方式

每次出现以下情况必须更新：

1. 同类错误第二次发生；
2. 某故障可能导致数据丢失、越权或不可恢复；
3. Agent 对边界产生稳定误解；
4. 新增高风险组件或外部系统；
5. POC 推翻既有关键假设。

每条新增规则需要记录：

```text
规则编号
发现日期
关联 Issue
问题现象
根因
影响
新增规则
自动化检查
验证证据
```
